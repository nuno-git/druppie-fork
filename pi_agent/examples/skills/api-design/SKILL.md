# API Design

Follow REST API best practices when building HTTP APIs.

## Guidelines
- Use proper HTTP methods (GET, POST, PUT, DELETE, PATCH)
- Return appropriate status codes (200, 201, 204, 400, 404, 500)
- Use consistent JSON response envelopes
- Validate all input at the boundary
- Use plural nouns for resource endpoints
- Version the API (v1, v2)
- Include proper error messages with error codes
