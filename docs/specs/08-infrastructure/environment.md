## Application

```
ENVIRONMENT=development                  # production|staging|development
DEV_MODE=false                           # true bypasses Keycloak (dev-only)
INTERNAL_API_KEY=druppie-internal-secret-key  # CHANGE FOR PROD
USE_MCP_MICROSERVICES=true
```

Production refuses to start if `INTERNAL_API_KEY` or `SANDBOX_API_SECRET` are defaults.

## Druppie self-update (update_core_builder)

```
DRUPPIE_REPO_OWNER=nuno-git
DRUPPIE_REPO_NAME=druppie-fork

GITHUB_APP_ID=
GITHUB_APP_PRIVATE_KEY_PATH=/app/secrets/github-app-private-key.pem
GITHUB_APP_INSTALLATION_ID=
```

Place the PEM at `secrets/github-app-private-key.pem` on the host; it's mounted RO.

## Docker socket (rootless Docker)

```
DOCKER_SOCK=/run/user/1000/docker.sock   # rootless: XDG_RUNTIME_DIR
```

Default `/var/run/docker.sock` works for most setups.

## Env var propagation

- Docker Compose reads `.env` automatically.
- Frontend Vite env vars (`VITE_*`) are read from `.env` by docker-compose and passed as build args to `druppie-frontend` / as env to `druppie-frontend-dev`.
- Backend reads env vars at startup via `pydantic-settings` in `druppie/core/config.py` (if present) or via `os.environ` lookups.
- MCP servers receive their own env subset via docker-compose.

## Secret rotation

No automatic rotation today. To rotate:
1. Update `.env`.
2. For Keycloak/Gitea secrets: also update via their admin UIs or re-run init with the new values.
3. Restart affected services.

## Secrets folder

`secrets/` is mounted read-only into the backend at `/app/secrets/`. Structure today:
- `github-app-private-key.pem` — PKCS#8 private key (not checked in).

Additions go here with appropriate file permissions (600 on host).
