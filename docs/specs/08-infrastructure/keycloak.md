# Keycloak

Identity provider. Runs `quay.io/keycloak/keycloak:24.0` in `start-dev` mode on port 8180 (external) / 8080 (internal).

Realm: `druppie`.

## Realm config (`iac/realm.yaml`)

Settings:
- `registrationAllowed: false`.
- `resetPasswordAllowed: true`.
- `rememberMe: true`.
- `loginWithEmailAllowed: true`.
- `accessTokenLifespan: 3600` (1 hour).
- `ssoSessionIdleTimeout: 1800` (30 min).
- `ssoSessionMaxLifespan: 36000` (10 hours).
- Brute force: enabled, `maxFailureWaitSeconds: 900`, `minimumQuickLoginWaitSeconds: 60`, `failureFactor: 5` (lock after 5 failures).
- Password policy: `length(8) and upperCase(1) and lowerCase(1) and digits(1)`.

## OAuth2 clients

### `druppie-frontend`
- Public client (no secret).
- Standard flow enabled.
- Redirect URIs: `http://localhost:5273/*`, `http://${EXTERNAL_HOST}:${FRONTEND_PORT}/*`.
- PKCE S256 enforced.

### `druppie-backend`
- Confidential.
- `serviceAccountsEnabled: true` (for service-to-service calls).
- Secret: `KEYCLOAK_CLIENT_SECRET` (generated by init, written to `.env`).

### `gitea`
- Confidential.
- Redirect URIs: `http://localhost:3100/*`, `http://${EXTERNAL_HOST}:${GITEA_PORT}/*`.
- Secret: `GITEA_CLIENT_SECRET`.
- Used by Gitea's OAuth2 provider integration â€” users click "Sign in with Keycloak" in Gitea.

## Setup script

`scripts/setup_keycloak.py`:
1. Authenticates to Keycloak master realm (admin-cli). Retries 10Ă— over 50s during startup.
2. Creates `druppie` realm (idempotent â€” 409 if exists).
3. Creates roles from `iac/realm.yaml` (composites supported).
4. Creates OAuth2 clients; generates secrets; writes to `.env`.
5. Creates users from `iac/users.yaml` with role assignments.

## Admin login

Master realm admin: `KEYCLOAK_ADMIN=admin` / `KEYCLOAK_ADMIN_PASSWORD=admin`. Change in `.env` for production.

Admin console: `http://localhost:8180/admin/` â†’ select `druppie` realm.

## Production requirements

- Override all default secrets (KEYCLOAK_ADMIN_PASSWORD, KEYCLOAK_CLIENT_SECRET, GITEA_CLIENT_SECRET, GITEA_SECRET_KEY).
- Run Keycloak in `start` mode (not `start-dev`) with `KC_HOSTNAME=<public domain>`, TLS.
- Back by a proper PostgreSQL (not single-container).
- Consider Keycloak HA for zero-downtime.
